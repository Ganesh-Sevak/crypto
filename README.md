# Crypto Market Risk & Liquidity Monitoring System

Real-time dashboard for monitoring simulated crypto market data, market risk, liquidity quality, SMS price alerts, and alert conditions across `BTC-USD`, `ETH-USD`, and `SOL-USD`.

## Architecture

The backend is a FastAPI service that simulates exchange ticks, computes rolling metrics, evaluates rule-based risk alerts, evaluates SMS price-alert subscriptions, and publishes updates over WebSockets. REST endpoints expose health, symbols, latest metrics, recent alerts, and SMS alert subscriptions. Redis is used when available for shared state and alert history; the service automatically falls back to an in-memory cache when Redis is not reachable.

Default data source: simulated market data generated by `backend/app/stream.py`. It is useful for demos and engineering tests. The backend also includes a public Coinbase Exchange ticker provider for real market checks and optional live streaming.

The frontend is a Vite React dashboard that consumes the WebSocket stream, renders live metric cards and a price chart, and includes a mock mode so the UI remains demo-ready if the backend is unavailable.

The `cpp/` folder contains an optional C++ volatility example that can be compiled as either a standalone executable or shared library callable from Python.

## Tech Stack

- Backend: Python, FastAPI, WebSockets, pandas, numpy
- Cache / broker: Redis with in-memory fallback
- Frontend: React, Vite, modern CSS
- Testing: pytest
- Containerization: Docker and docker-compose
- Optional performance module: C++17

## Features

- Simulated real-time streams for `BTC-USD`, `ETH-USD`, and `SOL-USD`
- WebSocket market updates for low-latency UI refreshes
- Metrics: volatility, bid-ask spread, VWAP, rolling price change, liquidity score, volume imbalance
- Rule-based alerts for high volatility, wide spreads, liquidity drops, and sudden volume imbalance
- SMS target-price alerts with Twilio support and safe dry-run mode
- One-click live Coinbase market price checks from the dashboard
- Optional live stream mode with `CRYPTO_MARKET_DATA_SOURCE=live`
- REST API:
  - `GET /health`
  - `GET /market-data/source`
  - `GET /market-data/live/{symbol}`
  - `GET /metrics`
  - `GET /alerts`
  - `GET /symbols`
  - `GET /sms/price-alerts`
  - `POST /sms/price-alerts`
- Structured JSON-style backend logging
- Frontend mock data mode
- Dockerized backend, frontend, and Redis

## Run Locally

From this directory:

```bash
cd ~/Desktop/crypto/crypto
```

Backend:

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

This project venv lives at `~/Desktop/crypto/crypto/backend/.venv`. On this machine it was created with Homebrew Python at `/opt/homebrew/opt/python@3.14/...`.

Redis is optional for local development. If Redis is not running, the backend logs a warning and uses in-memory state.

To stream public Coinbase ticker data instead of simulated ticks:

```bash
export CRYPTO_MARKET_DATA_SOURCE=live
```

The dashboard also has a **Check Live Price** button that fetches a real Coinbase quote for the selected symbol even when the main stream is simulated.

SMS alerts run in dry-run mode by default. Dry-run mode creates and triggers alert subscriptions but logs the message instead of sending a real text.

To send real SMS messages with Twilio, set these before starting the backend:

```bash
export CRYPTO_SMS_DRY_RUN=false
export CRYPTO_TWILIO_ACCOUNT_SID="your_account_sid"
export CRYPTO_TWILIO_AUTH_TOKEN="your_auth_token"
export CRYPTO_TWILIO_FROM_NUMBER="+15551234567"
```

Phone numbers must use E.164 format, for example `+13125551234`.

Frontend in a second terminal:

```bash
cd ~/Desktop/crypto/crypto/frontend
npm install
npm run dev
```

Open:

```text
http://localhost:5173
```

## Run With Docker

```bash
cd ~/Desktop/crypto/crypto
docker compose up --build
```

Services:

- Frontend: `http://localhost:5173`
- Backend API: `http://localhost:8000`
- Redis: `localhost:6379`

For Docker live mode, edit `docker-compose.yml` and set:

```yaml
CRYPTO_MARKET_DATA_SOURCE: live
```
